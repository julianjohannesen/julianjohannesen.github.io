What exactly are Git and GitHub?

Git is a piece of software that allows you to keep track of changes you and other people make to your files. Most importantly, it allows any number of people work on different parts of a complex project at the same time. New versions of a file can be branched off of a trunk , and those branches can later be merged back into the trunk. Any number of people anywhere in the world can work on versions of the same file, notifying administrators when they think their code is ready to be merged back into the master branch. It’s a great way for a group of people to work on a project together. Git is primarily used for source code management in software development , but it can be used to keep track of changes in any set of files. You can learn more by visiting the Git project’s documentation .

GitHub is a web-based hosting service for version control using git. It is mostly used for source code . It offers all of the distributed version control and source code management functionality of Git as well as adding its own features. It provides access control and collaboration features such as bug tracking , feature requests , task management , and wikis for every project.

What’s the command line?

You usually use a mouse or your fingers to click or tap icons on your desktop or elsewhere and access files, folders, and programs. But that’s just one of the ways you can communicate with your computer. The command line or command line interface (CLI) is a text-based interface that developers use to more effectively and efficiently communicate with computers to accomplish a wide set of tasks. Other names for the command line are: cmd, CLI, prompt, console or terminal.

There are at two command line interfaces built into Windows 10. The first is the older command prompt (CMD) and the second is the newer and more powerful PowerShell (PSH). PSH can used to perform and automate tasks not just in Windows but in other Microsoft products as well. Both CMD and PSH use commands that are unique to Windows and that wouldn’t work on a Unix machine or Mac. This can be problematic, because most developers are accustomed to commands that work on both Unix and Mac machines. Don’t worry, there are several workarounds.

The command line can be “ Run as Administrator.” Running the command line as an administrator will allow you to do things like install and delete programs, things you wouldn’t be able to do as an ordinary user. As you set up your development environment, you will often need to run the command line as an administrator. Pay close attention when you’re doing this, because, as an administrator, you will be able to do things that might damage your files or your Windows installation.

There are several excellent free courses out there on how to use the command line. You might want to try Codecademy’s free course on using the Unix command line . The course covers a lot of basic commands that are regularly used by developers. Udacity , Rithm School , and Viking Code School also offer free courses on how to use the command line.

Once you’re familiar with the command line, you might want to investigate Git for Windows and ConEmu , which will improve your command line experience.

Why take a course on using the Unix command line, if I have a Windows machine? 

We mentioned earlier that when you install the GitHub desktop application, Git Bash is also installed. Git Bash is another CLI that packages a variety of utilities together. Git Bash understands many Unix commands and is the CLI you’ll use when performing most git tasks.

Step 3: Configure Git

Open PowerShell in admin mode. Now enter the following commands, replacing “John Doe” and “ johndoe@example.com ” with the username and email address you used when you created your GitHub account.

{% highlight ruby %} git config --global user.name "John Doe" git config --global user.email johndoe@example.com {% endhighlight %}

To see all of your settings enter:

{% highlight ruby %} git config --list {% endhighlight %}

How do I open PowerShell?

Three ways to open PowerShell

  1. In your taskbar’s search box, enter PowerShell. Click or tap on the PowerShell result. To launch as administrator, right-click (or tap and hold, if you use a touchscreen) on the PowerShell search result, and then click/tap on “Run as administrator.”
  2. Press the Win + R keys on your keyboard. Type PowerShell and press Enter or click/tap OK
  3. Open your taskbar settings and set “Replace Command Prompt with PowerShell…” to “on.” Close settings. Now press the Win + X keys on your keyboard and select “Windows PowerShell” or “Windows PowerShell (Admin).”

Step 4: Install Chocolatey

We’re going to use the command line to install an application called Chocolatey.

Wait a minute! What is Chocolatey?

Chocolatey is what’s called a “package manager” for Windows. You can think of it as an application that assists in installing and updating other applications. We’re going to use it to install another application called Meteor.js. More on that later.

Cut and paste the following command into the command line:

{% highlight ruby %} Set-ExecutionPolicy Bypass -Scope Process -Force; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1')) {% endhighlight %}

Answer “Yes” when prompted. When Chocolatey has finished installing, close and then reopen PowerShell in admin mode.

What if I get errors while installing?

It’s not uncommon to encounter errors while installing Chocolatey or other packages. Don’t worry and don’t be discouraged. Expect that you may have to Google various error messages and that this may take some time. If Googling doesn’t work, or the solution is confusing, reach out to the CodeBuddies Slack community.
